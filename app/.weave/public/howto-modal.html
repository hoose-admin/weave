How to use this dashboard

Lifecycle

  • Scratch pad — ideas filed via + New. Thin stubs; no body yet.
  • Backlog — promoted stubs with full Objective / Context / Acceptance Criteria.
  • Staging — refined by AI, awaiting your approval. No code written yet.
  • Stuck — agentic flow is blocked: needs additional information or an approval before work can continue.
  • Building — implementation in progress.
  • Validating — build complete; awaiting review / QA.
  • Complete — accepted. Auto-archived after 7 days.

What the skill does

The ticket-manager skill owns the full .tickets/ lifecycle — creating, refining, moving, and archiving tickets, and keeping TaskBoard.md in sync after every operation.

The dashboard UI and the AI agent write to the same ticket files — there is no separate state. Core ops: create-ticket, promote-from-scratch, refine-from-backlog, build-ticket, move-ticket, link-tickets.

Example ticket flow

  1. Click + New → stub lands in Scratch pad.
  2. promote TKT-NNN → AI greps codebase, writes body, moves to Backlog.
  3. pick up TKT-NNN → AI refines for build-readiness, moves to Staging. No code yet.
  4. Review the staged ticket, then: build TKT-NNN → moves to Building, implementation begins.
  5. Implementation done → ticket auto-moves to Validating.
  6. Accept → move TKT-NNN to complete.
  7. 7 days later → auto-archived to 7-archive.

Example prompts

  • create a ticket to [describe feature] Files a new ticket in backlog with full context gathered from the codebase.
  • promote TKT-NNN Expands a scratch stub into a full backlog ticket.
  • pick up TKT-NNN Refines a backlog ticket and moves to staging. Does NOT build — you must approve first.
  • build TKT-NNN Implements a staged ticket and moves it to validating when done.
  • what's on the board? Refreshes TaskBoard.md and shows the current state of all tickets.
  • move TKT-NNN to validating Manually advances (or reverses) a ticket to any lifecycle stage.
  • TKT-NNN depends on TKT-MMM Creates a dependency link visible in the graphs view.
⚠ Staging gate: pick up TKT-NNN refines and stages only — it does not write any code. Say build TKT-NNN (or ship it) to advance to building. This gives you a review window before implementation starts.
