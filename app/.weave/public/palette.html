design tokens

Defined in :root at the top of styles.css. Consumers reference via var(--token-name).

font families

Two families loaded via Google Fonts at the top of styles.css: Fira Code (default UI + display) and JetBrains Mono (used inline on a small number of rules, e.g. ADR domain tags). Both --font-mono and --font-display resolve to Fira Code — the dashboard is intentionally monospace-everywhere.

buttons

Single .btn family with BEM variants (TKT-227). All variants share font-family var(--font-mono), font-size var(--font-size-base) (12px), padding 8px 16px, radius var(--radius). 

<button class="btn">Default</button>
<button class="btn btn--primary">Primary</button>
<button class="btn btn--secondary">Secondary</button>
<button class="btn btn--danger">Danger</button>
<button class="btn btn--transition">→ Transition</button>
<button class="btn" disabled>Disabled</button>

close-btn

Canonical dismissal control for modals (TKT-229). Lives inside .modal-head. Replaces the legacy .modal-close and .adr-modal-close variants.

Modal title 
<button type="button" class="close-btn" aria-label="Close">×</button>

toolbars

Single .toolbar primitive with page-class ancestor scoping (TKT-230). The body's page class (.board-page, .adr-page, .ticket-page, .graphs-page) selects per-page styling.

As seen on the board page (non-sticky, --bg background):

<body class="board-page">
  <div class="toolbar">
    <label>sort</label>
    <select>…</select>
  </div>
</body>

accordions

Canonical collapsible chrome used by the ADR and ticket detail views. Class structure: .adr-section.adr-section-<slug> + .adr-section-h. Nested subsections use .adr-subsection.

TL;DR

One-sentence statement of the decision.

</details>

status badges

Per-status pills shared by the ADR list, detail pages, and ticket headers. Class scheme: .adr-status + .adr-status-<state>.

proposed accepted rejected superseded deprecated 
<span class="adr-status adr-status-accepted">accepted</span>

modals

The dashboard ships two modal bases — legacy <div class="modal"> (toggled via the hidden attribute) and modern <dialog> (toggled via showModal()). Migration to single base is deferred. Both share the close-X primitive above.
